Modules are pre-integrated app features that compose with the ds-start foundation. Each module brings its own files, dependencies, and environment variables while following the same project conventions.

## Available Modules

### Data & Auth

| Module | Description | Prerequisites |
|--------|-------------|---------------|
| [Prisma](/docs/modules/prisma) | Prisma 6 ORM with PostgreSQL | None |
| [Better Auth](/docs/modules/auth) | Email/password + OAuth authentication | Prisma |
| [Clerk](/docs/modules/clerk) | Managed authentication with pre-built UI | None |
| [Stripe](/docs/modules/stripe) | Subscription billing and payments | Auth or Clerk |

### Features

| Module | Description | Prerequisites |
|--------|-------------|---------------|
| [Email](/docs/modules/email) | Transactional email with Resend | None |
| [File Uploads](/docs/modules/file-uploads) | S3-compatible file uploads | None |
| [Zustand](/docs/modules/zustand) | Client state management | None |
| [Forms](/docs/modules/forms) | JSON-driven form renderer | None |

### DevOps

| Module | Description | Prerequisites |
|--------|-------------|---------------|
| [GitHub Workflows](/docs/modules/github-workflows) | CI pipeline with GitHub Actions | None |
| [Vercel Deploy](/docs/modules/vercel-deploy) | CD pipeline for Vercel | GitHub Workflows |

## Adding Modules

Select modules through the interactive wizard when running `npx ds-start init`, or specify them with flags:

```bash
npx ds-start init my-app --prisma --auth --stripe
```

Use `--base` to scaffold the foundation without any modules, then add them later:

```bash
npx ds-start add email
```
